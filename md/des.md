# 부동산 경매 AI 튜터 플랫폼 화면 디자인 명세서 (des.md)

**디자인 에이전트 (`@des`) 작성**

본 문서는 부동산 경매 교육 플랫폼의 프리미엄 UI/UX 테마, 전역 CSS 커스텀 스타일시트, 반응형 미디어 쿼리 설계 사상 및 핵심 컴포넌트별 렌더링 명세를 정리한 디자인 사양서입니다.

---

## 1. 디자인 시스템 및 전역 CSS 정의 (`src/core/common.py`)

본 플랫폼은 Streamlit의 기본 레이아웃이 지닌 한계를 극복하고, 학습자에게 프리미엄하고 통일감 있는 사용자 경험(UX)을 선사하기 위해 `src/core/common.py` 모듈 내에 전역 커스텀 CSS (`_GLOBAL_CSS`)를 주입하여 화면 전체의 스타일을 제어합니다.

### 1.1 핵심 컬러 시스템 (CSS Variables)
다크 모드와 라이트 모드 요소가 세련되게 공존하는 프리미엄 테마를 표현하기 위해 고유의 7가지 HSL/Hex 테마 색상을 사용합니다.

```css
:root {
    --primary: #2563eb;      /* 신뢰감을 주는 비비드 블루 (주요 강조) */
    --secondary: #0d9488;    /* 전문적인 느낌의 딥 틸/청록 (서브 강조 및 브랜드 컬러) */
    --accent: #fb7185;       /* 주목도를 높이는 소프트 로즈 (특수 권리 등 경고) */
    --warning: #f59e0b;      /* 시선을 끄는 오렌지 옐로우 (면책 조항 경고) */
    --dark: #0f172a;         /* 깊이감 있는 슬레이트 블랙 (헤더 및 주요 정보 배경) */
    --light: #f8fafc;        /* 산뜻하고 깨끗한 화이트 그레이 (전체 배경 및 빈 영역) */
    --success: #10b981;      /* 합격/정답의 에메랄드 그린 (피드백 결과 완료) */
}
```

### 1.2 서체 및 폰트 시스템
가독성을 높이고 법률 용어의 무거운 느낌을 중화하기 위해 현대적인 다국어 서체를 구글 폰트(`Google Fonts`)로 로드하여 적용합니다.
- **선언 코드**: `@import url('https://fonts.googleapis.com/css2?family=Inter:wght@400;600;700;800&family=Noto+Sans+KR:wght@300;400;500;700;900&display=swap');`
- **적용 대상**: `html`, `body`, 마크다운 텍스트 영역 전체에 `'Noto Sans KR'`, `'Inter'`, `sans-serif` 순으로 우선순위 바인딩.

### 1.3 기본 레이아웃 및 내비게이션 숨김 정책
- **사이드바 제거**: Streamlit의 기본 왼쪽 사이드바 내비게이션 리스트는 메뉴가 좌측에 길게 늘어져 있어 가시성을 저해하므로 아래 CSS 오버라이드로 완전히 숨김 처리합니다.
  ```css
  [data-testid="sidebar-nav-container"], [data-testid="stSidebarNav"] {
      display: none !important;
  }
  ```
- **페이지 레이아웃**: 모든 페이지는 `st.set_page_config`를 통해 `layout="wide"` 및 `initial_sidebar_state="collapsed"`로 초기화하여 가로 해상도를 최대로 확보합니다.

## 2. 글로벌 공통 컴포넌트 디자인 상세

### 2.1 상단 가로형 네비게이션 (GNB TOP Menu)
사이드바를 숨기는 대신, 최상단에 웹 포털 스타일의 세련된 가로형 메뉴(GNB)를 배치했습니다.
- **구조**: `st.columns`를 활용한 7개 열 레이아웃 구조 (`[2.0, 2.0, 1.9, 1.9, 1.9, 2.1, 0.6]`)
- **디자인 피드백 (Hover & Active)**: 
  - 기본 상태는 테두리(`border: 1.5px solid #cbd5e1`), 배경(`white`), 텍스트 색상(`slate-900`)을 유지합니다.
  - 마우스 호버(`hover`) 시 입체적인 카드 상승 효과(`transform: translateY(-3px)`)와 소프트 그라데이션 배경(`linear-gradient(135deg, rgba(37,99,235,0.04) 0%, rgba(13,148,136,0.06) 100%)`)을 반영하여 세련된 피드백을 전달합니다.
- **반응형 텍스트 스케일링**: 미디어 쿼리를 도입하여 화면 해상도에 맞게 폰트 크기를 유기적으로 축소합니다.
  - **1600px 미만**: GNB 폰트 크기 `1.3rem`, padding `8px 14px`로 스케일 다운.
  - **1200px 미만**: GNB 폰트 크기 `1.15rem`, padding `6px 12px`로 스케일 다운.
  - **768px 미만**: GNB 폰트 크기 `0.95rem`, padding `5px 10px`로 모바일 가독성 확보.

### 2.2 그라데이션 히어로 배너 (`.hero-container`)
각 화면의 최상단에서 해당 기능의 목적을 직관적으로 소개하는 브랜드 영역입니다.
- **배경**: `#f1f5f9`에서 `#ffffff`로 이어지는 135도 은은한 그라데이션.
- **좌측 포인트 바**: `--secondary` (#0d9488) 색상의 두꺼운 좌측 보더(`border-left: 6px solid`)가 있어 시각적 집중도를 부여합니다.
- **타이틀 텍스트 (`.hero-title`)**: 2.5rem의 아주 굵은 폰트 크기(weight: 900)에 그라데이션 마스크(`-webkit-background-clip: text`)를 씌워 텍스트 자체에 고급스러움을 가미합니다.

### 2.3 홈페이지형 퀵 메뉴 카드 (`.card-container`)
메인 화면(`app.py`)에서 각 주요 기능으로 이동을 유도하는 디자인 카드입니다.
- **디자인 사양**: 흰색 배경, 라운드 코너(`border-radius: 20px`), 미세 그림자(`box-shadow`).
- **마우스 호버**: 카드가 약간 떠오르고(`transform: translateY(-5px)`), 테두리 선 색상이 주 청록색(`--secondary`)으로 교체되며 반응형 음영이 짙어집니다.

### 2.4 에이전트 전용 HTML 메타 배지
채팅 UI에서 답변 주체(에이전트)를 표시하는 맞춤형 소형 배지 컴포넌트입니다.
- **경매 절차 에이전트 (`procedure`)**: 소프트 블루 배경 (`rgba(37,99,235,0.08)`), 블루 보더, 이모지 `📋`.
- **권리분석 튜터 에이전트 (`tutor`)**: 소프트 청록 배경 (`rgba(13,148,136,0.08)`), 청록 보더, 이모지 `📚`.
- **사례 퀴즈 채점 에이전트 (`quiz`)**: 소프트 로즈 배경 (`rgba(225,29,72,0.08)`), 로즈 보더, 이모지 `📝`.

### 2.5 프리미엄 데이터 테이블 (`.premium-table`)
매각물건명세서의 등기부 현황, 임차인 정보 등 건조한 표 데이터를 프리미엄하게 가공합니다.
- **테두리**: `border-radius: 12px`, `overflow: hidden`을 통해 둥글게 감싸진 컨테이너 형태.
- **헤더 (`th`)**: 은은한 슬레이트 그레이 배경(`#f1f5f9`), 굵은 폰트 세팅.
- **행 호버 (`tr:hover`)**: 마우스를 올린 행의 배경색이 미세하게 변화(`#f8fafc`)하여 아이트래킹을 돕습니다.

## 3. 페이지별 핵심 레이아웃 및 시각화 명세

### 3.1 메인 홈 화면 (`app.py`)
- **구조**: 상단 히어로 배너 ➔ 4단계 멀티에이전트 워크플로우 맵 ➔ 4열 그리드 배치(절차 안내, 권리분석 튜터, 사례 퀴즈, 용어 사전) ➔ 최하단 공통 디스클레이머.
- **워크플로우 맵**: 각 에이전트의 연계 과정을 시각화하기 위해 가로형 흐름 화살표(`.flow-arrow-horizontal`)를 배치하고, 태블릿 이하 해상도(992px 미만)에서는 화살표가 90도 회전하여 아래 방향으로 자연스럽게 세로 정렬되도록 미디어 쿼리를 구성했습니다.

### 3.2 경매 절차 안내 (`pages/auction_procedure.py`)
- **구조**: `[3:2]` 비율의 2열 레이아웃.
- **왼쪽 영역 (Q&A 챗봇)**: 질문을 유도하는 입력 필드 및 자주 묻는 질문 퀵버튼을 바둑판식으로 정렬 배치.
- **오른쪽 영역 (9단계 타임라인)**: `st.expander`를 통해 9단계별 아코디언 일람을 구축하여, 복잡한 내용을 사용자가 원하는 단지만 펼쳐볼 수 있게 깔끔하게 압축.

### 3.3 권리분석 튜터 (`pages/rights_analysis.py`)
- **구조**: `[3:2]` 비율의 2열 레이아웃.
- **오른쪽 영역 (권리분석 핵심 족보)**: 튜터의 소크라테스 대화 도중 핵심 원칙을 상시 대조할 수 있도록, `st.info` (말소기준권리), `st.warning` (임차인 대항력), `st.error` (항상 인수되는 권리) 컴포넌트를 사용하여 위험도와 경중을 컬러 매칭으로 레이아웃 구성.

### 3.4 사례 퀴즈 연습 (`pages/case_quiz.py`)
- **구조**: 단일 열(Full Width) 집중형 레이아웃.
- **물건지 명세 구조**: 
  - 실물 물건 정보: 다크 블루 컬러 패널 (`#0F172A`)을 사용하여 한눈에 중요 자산 가치(감정가, 최저매각가)를 부각.
  - 등기사항전부증명서 및 임차인 현황: `.premium-table`로 구현되어 복잡한 연도와 권리 내용을 흐트러짐 없이 구조화.
  - 특수 권리: 배경에 옅은 빨강(`rgba(239, 68, 68, 0.05)`) 및 빨간색 알림 배지를 얹어 시각적 긴장도 조성.
- **채점 피드백 박스**: 채점이 완료되면 AI의 분석과 피드백 항목이 가로 구분선이 있는 어두운 슬레이트 카드 컨테이너 내부에 정렬되어 표시.

### 3.5 경매 용어 사전 (`pages/glossary.py`)
- **초성 필터 버튼바**: 한글 14개 자음과 '전체' 옵션을 포함한 15개 열의 가로 정렬 버튼바 구성.
  - 사용자가 선택한 초성 버튼은 볼드 텍스트(`**ㄱ**`)로 실시간 시각적 활성화 표시.
- **용어 그리드 카드**: 가나다 정렬된 용어에 맞게 카드가 배치되며, 하단 영역에 점선으로 구분된 메타 정보(`관련 조문 근거`, `초성 배지`)를 배치해 사전 디자인 완성도 극대화.

### 3.6 AI 입찰가 예측 (`pages/bid_prediction.py`)
- **입력 폼 레이아웃**: 기본 정보(셀렉트 박스) ➔ 가격 정보(원화 단위를 감안한 만원 단위 숫자 입력 필드) ➔ 대출 비율 및 금리(슬라이더) ➔ 추가비용(가로 3열 배치)의 논리적인 세로 분할 구조.
- **지표 대시보드 (Metrics)**: 상단에 입찰 마지노선, 예상 경쟁자 수, 낙찰 성공 확률 메트릭 카드를 나란히 배치하여 핵심 성과 지표(KPI) 형태로 낙찰가 시뮬레이션 결과를 요약 노출.
- **확률 분포 차트 (Probability Distribution Chart)**: `st.bar_chart`를 연동하고 브랜드 컬러인 청록색(`--secondary` / `#0d9488`) 테마로 입찰가 구간에 따른 낙찰 성공 확률 곡선을 막대그래프로 렌더링.
- **AI 입찰 전략 카드**: 연한 에메랄드와 블루의 혼합 그라데이션 패널과 청록색 좌측 보더를 주입해, AI가 처방하는 정성적 분석글의 가독성과 주목도를 강화.
