'수정 계획서: implementation_plan.md, 작업 진척 사항: task.md, 최종 변경 이력: walkthrough.md' 한글로 작성해줘.

@pla './md/project0.md', './md/project1.md' 모든 내용을 포함 시켜주고 중복만 제거해서 README.md 생성해줘.

GEMINI : 'README.md' 첨부파일 기준으로 기획자료 만들어줘.
GEMINI : '[프로젝트 기획서] 부동산 경매 교육 멀티에이전트 RAG', 'README.md' 첨부파일 기준으로 디자인 및 화면 UI 기획안 만들어줘.

@pla 'README.md' 분석하여 프로젝트 디렉토리 및 소스 코드 생성해줘.

@des 웹 페이지 화면를 홈페이지 처럼 구성해줘. 기능을 메뉴별로 페이지 따로 만들어서 구성하고 메뉴별 디렉토리와 파일을 관리 할 수 있도록 해줘.

@des 왼쪽에 있는 메뉴를 TOP 메뉴로 변경해줘.

@des TOP 메뉴 폰트 크기를 현재 보다 크게해서 일반 홈페이지의 TOP 메뉴 처럼 보이게 해줘. 필요하면 tailwind 를 사용해서 이쁘게 구성해줘.

@des TOP 메뉴 폰트 크기가 3배 크게로 변경되지 않고 있어. 필요하면 tailwind 를 사용해서 적용.

@des '🏛️ 경매 AI 튜터 홈, 📋 경매 절차 안내, 📚 권리분석 튜터, 📝 사례 퀴즈 연습, 🔍 경매 용어 사전' 화면 제일 위에 위치한 해당 영역을 TOP 메뉴 라고 해. 해당 메뉴 텍스트 폰트 크기가 작아서 폰트 크기를 크게 만드려고 해. 적용이 안되고 있어. 현재 소스 코드의 app.py 코드도 검토하고 수정해줘. @dev 도 검토해줘.

@dev 코드에서 사용하는 설치해야 할 패키지들을 정리해서 requirements.txt 파일로 정리해줘. 그리고 pyproject.toml 파일도 생성해줘.

프로젝트를 분석하여 각 에이전트의 md 파일을 만들어줘.
@pla 기획서 작성 전문가 에이전트 ./md/pla.md 파일 만들기.
@dev 코드 개발 전문가 에이전트 ./md/dev.md 파일 만들기.
@des 화면 디자인 저문가 에이전트 ./md/des.md 파일 만들기.


### PPT 파일 생성

4개의 md 파일을 참고하여 PPT 파일 생성.

[페이지 별 적용해야 할 내용]

1. 표시 및 제목
2. 목차
3. 프로젝트 개요
4. 개발 참여자 (4명)
5. 개발 환경 및 아키텍처
6. 주요 기능별 화면 캡쳐 (5개 페이지)
7. 차별화
8. 마무리

[디자인 사항]

- 전체적 화면은 깔끔한 템플릿 스타일 유지
- 폰트는 PPT 파일 또는 구글 슬라이드에서 사용해도 문제가 없는 깔끔한 폰트로 지정.
- 오른쪽 아래 페이지 번호는 표시. (예 : 1/9, 2/9, 3/9, ... 9/9)
