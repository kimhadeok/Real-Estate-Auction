# **\[기획안\] AI Agent 기반 차세대 부동산 경매 플랫폼: 'AuctionAgent AI'**

## **1\. 프로젝트 개요 (Overview)**

### **1.1. 기획 배경**

- **기존 경매 플랫폼의 한계**: 단순히 법원 정보를 긁어와 보여주거나 정형화된 유료 권리분석 정보만 제공함. 사용자는 여전히 수많은 문서를 직접 대조하고, 발품(임장)을 팔며, 리스크를 감수해야 함.
- **AI Agent 기술의 도래**: LLM(대형 언어 모델)과 AI Agent(자율 에이전트)의 발전으로 복잡한 비정형 문서 분석(매각물건명세서, 현황조사서 등), 실시간 웹 데이터 그라운딩(시세, 지역 호재 검색), 맞춤형 협상 전략 수립이 가능해짐.
- **기획 방향**: 복잡하고 두려운 경매 투자를 "AI 비서와 함께하는 안전하고 스마트한 투자"로 패러다임을 전환하여, 경매 초보자의 진입 장벽을 낮추고 고수들의 업무 효율을 10배 이상 극대화함.

### **1.2. 서비스 슬로건**

**"복잡한 권리분석부터 현장 임장, 명도 협상까지 \- 당신만을 위한 경매 AI 에이전트가 움직입니다."**

### **1.3. 핵심 타겟 페르소나**

- **초보 투자자 (민수 씨, 30대 후반)**: "내 집 마련을 경매로 하고 싶은데, 혹시 보증금을 날릴까 봐 너무 무서워요. 권리분석과 명도가 가장 걱정입니다."
- **프로 전업 투자자 (정희 씨, 50대 초반)**: "매주 전국 수십 개의 물건을 분석하느라 시간이 너무 부족해요. 가상으로 임장 리포트를 뽑아주고, 낙찰가율을 정확히 예측해 줄 도구가 필요합니다."

## **2\. 'AuctionAgent AI' 핵심 기능 정의 (Core Features)**

본 서비스는 **4가지 특화된 AI 에이전트**가 협업하는 **'Multi-Agent System'** 구조를 가집니다.

\[사용자 질의/물건 선택\]  
 │  
 ├───► ① AI 권리분석 에이전트 (법률·등기 분석 및 인수 리스크 경고)  
 ├───► ② AI 입지·가상임장 에이전트 (실시간 시세 그라운딩, 호재 분석, 가상 임장)  
 ├───► ③ AI 입찰가 예측 에이전트 (최근 낙찰 데이터 기반 적정가 시뮬레이션)  
 └───► ④ AI 명도·법률 코치 에이전트 (점유자 성향 분석, 시나리오별 협상 시뮬레이터)

### **2.1. AI 권리분석 에이전트 (Eagle-Eye Explorer)**

_가장 핵심적인 사법 리스크를 제거하는 분석 엔진입니다._

- **기능 설명**: 등기부등본, 매각물건명세서, 현황조사서 등 법원 제공 비정형 문서를 실시간 OCR 및 언어 모델로 분석하여 권리 관계의 안전성을 도출합니다.
- **세부 액션**:
  - **말소기준권리 자동 지정**: 모든 권리의 소멸 여부를 시각적 타임라인으로 제공.
  - **인수 권리 경고**: 유치권, 법정지상권, 대항력 있는 임차인의 보증금 인수 여부 등 '돈이 추가로 들어가는 리스크'를 빨간색 경고등으로 표시.
  - **보증금 인수액 시뮬레이터**: 임차인의 배당 요구 여부와 확정일자를 계산하여, 낙찰자가 추가로 인수해야 할 실질적인 금액을 1원 단위까지 추정.

### **2.2. AI 입지 및 가상 임장 에이전트 (Virtual Reporter)**

_현장에 가지 않고도 80%의 정보를 손에 쥐는 데이터 그라운딩 에이전트입니다._

- **기능 설명**: Google Search Grounding 및 국토교통부 실거래가 API, 네이버 부동산 매물 API 정보를 융합하여 실시간 가치를 평가합니다.
- **세부 액션**:
  - **가상 임장 브리핑**: "해당 아파트단지 정문에서 초등학교까지 도보 5분 소요되나 언덕길이 있습니다.", "최근 엘리베이터 교체 공사가 있었습니다" 등 실제 후기 분석 보고서 생성.
  - **실시간 호재/악재 스크리닝**: 대상 물건 반경 2km 이내의 교통 호재(지하철 연장 등), 재개발 계획, 혐오시설 유무를 뉴스 및 도시계획 포털에서 실시간으로 긁어와 요약.
  - **급매가 대조 분석**: 현재 등록된 인근 공인중개업소 최저가(급매)와 경매 감정가를 대조하여 메리트율(%) 도출.

### **2.3. AI 입찰가 예측 에이전트 (Price Oracle)**

_수익성은 극대화하고 단독 낙찰(오버 페이)은 피하는 머신러닝 기반 시뮬레이터입니다._

- **기능 설명**: 해당 법원의 최근 3년간 낙찰가율 데이터, 물건 종류별 경쟁률, 현 시점 주택담보대출 금리 추이, KB 시세 등을 종합 분석하여 입찰 성공 확률 분포를 제안합니다.
- **세부 액션**:
  - **목표 수익률 기준 입찰가 제안**: 사용자가 "세후 2천만 원 수익을 내고 싶어"라고 입력하면 취등록세, 법무비용, 명도비, 인테리어비, 대출 이자를 역산하여 최대 입찰 마지노선을 제안.
  - **경쟁자 예측 수치**: 예상 입찰 인원수(예: 3\~5명 예상) 및 낙찰 확률이 가장 높은 구간을 그래프로 시각화.

### **2.4. AI 명도 및 법률 코치 (Negotiation Sensei)**

_초보자들이 가장 두려워하는 낙찰 이후의 '사람 상대(명도)' 프로세스를 돕는 비서입니다._

- **기능 설명**: 점유자(소유자, 임차인)의 유형에 따른 맞춤형 대화법, 이사비 산정 가이드, 법적 강제집행 절차 시나리오를 설계합니다.
- **세부 액션**:
  - **대화 시뮬레이터 (AI 롤플레잉)**: 감정이 격앙된 점유자 역할을 AI가 수행하고, 사용자가 타이핑하며 협상을 연습할 수 있는 시뮬레이션 인터페이스 제공. (예: "이사비 1,000만 원 안 주면 절대 안 나갑니다\!"라는 AI에 대처하는 현명한 답변 코칭)
  - **자동 서식 생성**: 인도명령 신청서, 내용증명(우편 발송용), 점유이전금지가처분 신청서 등 필요한 법적 서식의 필수 인적 사항과 내용을 즉시 완성형 PDF로 다운로드 지원.

## **5\. 기술 아키텍처 및 데이터베이스 설계전략 ()**

경매 서비스의 특성상 초기 서비스 론칭 단계에서의 가성비와 속도를 챙기면서도, 유입자가 많아지는 비즈니스 성장기에 시스템을 원활하게 이전할 수 있도록 **SQLite \+ ChromaDB** 조합의 아키텍처를 적용합니다.

### **5.1. AI 에이전트 파이프라인**

1. **RAG (Retrieval-Augmented Generation)**:
   - 대한민국 대법원 판례 데이터베이스를 임베딩하여 로컬 **ChromaDB**에 저장.
   - 사용자가 "유치권 성립 요건이 어떻게 되나요?"라고 질문하면, ChromaDB에서 관련 판례 벡터를 조회하여 컨텍스트를 가져온 뒤 gemini-2.5-flash-preview-09-2025 API를 통해 정확한 법적 가이드를 생성.
2. **Google Search Grounding**:
   - 실시간 급매물 시세, 지역 개발 호재, 네이버 부동산 매물 변동 상황 등 실시간 데이터를 수집하기 위해 Gemini API의 tools: \[{ "google_search": {} }\] 기능을 동적 활성화하여 할루시네이션(환각 현상) 방지.
3. **Structured Output (구조화된 JSON 출력)**:
   - Gemini API 호출 시 responseSchema를 활용해 정형화된 JSON 형태의 분석 리포트를 수신하여, 바닐라 자바스크립트가 복잡한 파싱 과정 없이 DOM을 직접 조작할 수 있도록 설계.

### **5.2. 기술 스택 예시**

- **Frontend**: **Vanilla HTML5, CSS3 (Tailwind CSS CDN 사용), Vanilla JavaScript (ES6+)**
  - 프레임워크가 주는 빌드 오버헤드가 없으므로 초기 로딩 속도가 극도로 빠르며 가볍게 구동됨.
- **Backend**: **FastAPI (Python)**
  - 파이썬 생태계의 머신러닝/AI 라이브러리(ChromaDB Client, Gemini SDK)와의 호환성이 매우 우수함.
- **Relational Database**: **SQLite**
  - 서버 설정이나 추가 비용 없이 단일 로컬 파일 형태로 완벽하게 구동 가능.
- **Vector Database**: **ChromaDB**
  - 로컬 인메모리 및 파일 형태로 즉시 실행 가능한 초경량 벡터 DB.

### **5.3. PostgreSQL 전환을 대비한 핵심 마이그레이션 아키텍처 설계**

추후 동시 접속자 수 증가 및 PostGIS(공간 데이터) 연동을 위해 PostgreSQL로 마이그레이션할 때 코드를 변경하지 않도록 아래 3대 규칙을 기반으로 구현합니다.

1. **레포지토리 패턴 (Repository Pattern)**:
   - 데이터베이스 읽기/쓰기 영역을 비즈니스 로직과 철저히 격리합니다.
   - 파이썬 백엔드 내에 데이터 액세스 추상 클래스(BaseRepository)를 만들고 이를 상속받은 SQLiteRepository를 구현합니다. 추후 DB를 이전할 때 비즈니스 코드 수정 없이 PostgreSQLRepository 클래스로 갈아 끼우기만 하면 완료됩니다.
2. **ID 체계의 통일 (UUID 기반 설계)**:
   - SQLite의 정수형 Auto-Increment ID 대신 고유 식별자인 **UUID**를 프라이머리 키로 지정합니다.
   - UUID를 사용하면 SQLite의 관계형 데이터 ID와 ChromaDB의 벡터 데이터 ID(ids=\[...\])를 완벽하게 일치시킬 수 있어, 추후 PostgreSQL의 관계형 테이블 및 pgvector 통합 테이블로 합칠 때 데이터 병합 오차가 발생하지 않습니다.
3. **느슨한 관계 결합 (Loose Coupling)**:
   - ChromaDB 벡터의 메타데이터에는 SQLite에 기록된 '경매 사건 고유 UUID'만 등록합니다.
   - 검색 시 ChromaDB 유사도 매칭을 먼저 실행하여 결과 UUID 목록을 추출한 다음, SQLite에서 SELECT \* FROM auctions WHERE id IN (UUID 목록) 구조로 정보를 로드합니다. 이 패턴은 추후 PostgreSQL로 이관되어도 완벽하게 동일하게 작동합니다.

## **7\. 단계별 개발 로드맵 (Roadmap)**

### **Phase 1: MVP (최소 기능 제품) 개발 (1\~3개월)**

- **프론트엔드**: 바닐라 HTML/CSS/JS 기반의 극도로 가볍고 빠른 반응형 웹 인터페이스 구축.

- **기능**: 등기부등본 및 매각물건명세서를 업로드하면 AI 권리분석 결과를 보여주고 요약해 주는 기본 웹 비서 기능 제공.

- PostGIS를 연동하여 반경 인프라 검색, 역세권 거리 계산 등 고도화된 정밀 공간 분석 서비스 출시.
